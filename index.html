Elliptic Curve Cryptography

The class project for EECS 475 is implementing a variant of an Elliptic Curve Cryptosystem based off of the Menezies-Vanstone Elliptic Curve Cryptosystem. The instructors provided a precision integer class called Uberzahl and I implemented the operations needed to do the calculations within the field by overloading operators. Some of the algorithms used included the Extended Euclidian Algorithm for calculating inverses, needed to implement the encryption and decryption algorithms.

  • Class: EECS 475 - Cryptography
  • Date: November 2015
  • Languages: C++

Revenge of the Living Deadline

Revenge of the Living Deadline was a command-line simulator that involved shooting arrows at zombies to kill them before they get too close to the player. The design of the game involved loading all of the zombies into a priority queue data structure that sorted them based off of how much health they had left and their proximity to the player. Priority queues automatically re-sort after a node is popped off from the structure, making this data structure appropriate for zombie killing. I implemented the prioity queue as well as the sorting algorithms and game logic.

  • Class: EECS 281 - Data Structures and Algorithms
  • Date: February 2014
  • Languages: C++

Blob and Edge Detection

In Computer Vision, a blob is an area of an image where some properties are constant or at least relatively constant. Blob detection, following from this, finds regions of images that differ in properties compared to other regions around them. These properties, for example, can be color or brightness. My implmentation included a Laplacian of Gaussian (LoG) filter convolved over the image. The approaches I used were to one: scale up the filter, and two: scale down the image for each convolution with the LoG filter. My finding showed that scaling down the image for each convolution was more efficient than scaling up the filter each time.

  • Class: EECS 442 - Computer Vision
  • Date: February 2015
  • Languages: MATLAB

Image Stitching

My implementation for an image stitching program involved detecting feature points in the images to be stitched, and then extracting the pixel values of small regions around them. These values were then used to calcuate "distance," or similarity, between the neighborhood regions around the feature points in both images. This was done using Euclidian distance between SIFT descriptors. The closest matches, feature point pairs with a distance under a certain threshold, were then used to calculate a homography using the RANSAC algorithm between two images that would transform the left side of the image. The transformed image and right side image were then warped together using MATLAB's imtransform function. My current implementation works on two images, I am currently working on an implementaion that works on multiple images automatically.

  • Class: EECS 442 - Computer Vision
  • Date: March 2015
  • Languages: MATLAB
